# 🎯 ЗАДАНИЕ: Система управления библиотекой

## Описание предметной области

Тебе нужно разработать REST API для городской библиотеки. Система должна управлять книгами, читателями и выдачей книг.

---

## Требования к функционалу

### 1. Управление книгами
- ❌ Добавление новой книги
- ❌ Просмотр всех книг
- ❌ Просмотр одной книги по ID
- ❌ Обновление данных книги
- ❌ Удаление книги
- ❌ Поиск книг по названию или автору

### 2. Управление читателями
- ❌ Регистрация читателя
- ❌ Просмотр всех читателей
- ❌ Просмотр одного читателя
- ❌ Обновление данных читателя
- ❌ Удаление читателя

### 3. Выдача книг
- ❌ Выдать книгу читателю
- ❌ Вернуть книгу
- ❌ Посмотреть историю выдач
- ❌ Посмотреть, какие книги у читателя сейчас

***

## Структура данных

### Таблица: Books (Книги)
| Поле | Тип | Ограничения |
|------|-----|-------------|
| id | INTEGER | PRIMARY KEY, автоинкремент |
| title | VARCHAR(200) | NOT NULL |
| author | VARCHAR(100) | NOT NULL |
| isbn | VARCHAR(13) | UNIQUE, NOT NULL |
| year | INTEGER | CHECK (year >= 1000 AND year <= 2100) |
| pages | INTEGER | CHECK (pages > 0) |
| available | BOOLEAN | DEFAULT TRUE |

### Таблица: Readers (Читатели)
| Поле | Тип | Ограничения |
|------|-----|-------------|
| id | INTEGER | PRIMARY KEY |
| full_name | VARCHAR(150) | NOT NULL |
| email | VARCHAR(100) | UNIQUE, NOT NULL |
| phone | VARCHAR(20) | |
| registration_date | DATE | DEFAULT сегодняшняя дата |

### Таблица: Loans (Выдачи)
| Поле | Тип | Ограничения |
|------|-----|-------------|
| id | INTEGER | PRIMARY KEY |
| book_id | INTEGER | FOREIGN KEY → books(id) |
| reader_id | INTEGER | FOREIGN KEY → readers(id) |
| loan_date | DATE | NOT NULL, DEFAULT сегодня |
| return_date | DATE | NULL (если книга не возвращена) |
| due_date | DATE | NOT NULL (срок возврата: +14 дней от loan_date) |

***

## Бизнес-правила (валидация)

### Правило 1: ISBN
- ISBN должен быть уникальным
- Состоит из 13 цифр
- Проверяй формат: только цифры

### Правило 2: Год издания
- Не может быть раньше 1000
- Не может быть больше текущего года + 1

### Правило 3: Выдача книги
- Книгу можно выдать только если `available = TRUE`
- При выдаче: `available` меняется на `FALSE`
- При возврате: `available` меняется на `TRUE`

### Правило 4: Срок возврата
- Автоматически: loan_date + 14 дней
- Если книга не возвращена в срок — пометить как просроченную

### Правило 5: Один читатель
- Может взять максимум 3 книги одновременно
- Проверяй при выдаче

## Принцип 4: Модели vs Схемы

**Понимай разницу:**

| | models.py | schemas.py |
|---|-----------|------------|
| **Для чего** | Описание БД | Валидация данных API |
| **Библиотека** | SQLAlchemy | Pydantic |
| **Когда используется** | При работе с БД | При получении/отправке JSON |
| **Пример** | `Column(Integer, primary_key=True)` | `id: int` |

**Правило:**
- `models.py` — как хранятся данные
- `schemas.py` — как передаются данные

## Принцип 6: Валидация на двух уровнях

**Проверяй данные дважды:**

1. **В schemas.py** (Pydantic):
   - Типы данных (int, str, date)
   - Обязательные поля
   - Формат (email, телефон)

2. **В models.py** (SQLAlchemy):
   - Бизнес-правила
   - Связи между таблицами
   - Сложная логика

**Пример:**

```python
# schemas.py — проверка формата
class BookCreate(BaseModel):
    isbn: str
    
    @validator('isbn')
    def check_isbn_format(cls, value):
        if len(value) != 13 or not value.isdigit():
            raise ValueError("ISBN должен содержать 13 цифр")
        return value

# models.py — проверка уникальности в БД
class Book(Base):
    isbn = Column(String(13), unique=True, nullable=False)
```

## Принцип 9: Обработка ошибок

**Возвращай понятные ошибки:**

```python
# ✅ ХОРОШО:
if not book:
    raise HTTPException(
        status_code=404, 
        detail="Книга с ID 5 не найдена"
    )

# ❌ ПЛОХО:
if not book:
    raise Exception("error")
```

**Статус-коды:**
- `200` — OK
- `201` — Created (после POST)
- `400` — Bad Request (неправильные данные)
- `404` — Not Found (не найдено)
- `500` — Internal Server Error (ошибка сервера)
